# NSA Design System

The shared visual language for North Shore Automation — colors, type, space, components, product accents — distributed as a single CSS file.

## Live reference site

<https://northshoreautomation.github.io/nsa-brand-design/>

Browse the tokens, components, patterns, and per-product themes (Akomi, Tsunami, The Watcher).

## Use it in another project

Add this line to `<head>` in any HTML, before your own styles:

```html
<link rel="stylesheet" href="https://northshoreautomation.github.io/nsa-brand-design/dist/nsa.css">
```

That single file loads IBM Plex Sans + Mono, Microgramma D Bold Extended, and the full token + component layer. No other font imports needed.

To make Claude follow the design system when generating HTML in another project, paste the contents of [`CLAUDE.md.snippet.md`](./CLAUDE.md.snippet.md) into that project's `CLAUDE.md`.

## Repository layout

| Path | Purpose |
|---|---|
| `tokens.css`, `components.css` | Source CSS (edit these) |
| `build.sh` | Regenerates `dist/nsa.css` (version from `$NSA_VERSION` or most recent `v*` tag) |
| `dist/` | Generated by `build.sh`; gitignored. Workflow builds fresh per push. |
| `site/` | Reference site (HTML pages) |
| `fonts/` | Microgramma TTFs (intentionally tracked) |
| `assets/` | Logo |
| `sidebar.js` | Shared sidebar; `__NSA_VERSION__` placeholder substituted by workflow |
| `CLAUDE.md` | Maintainer-facing context for Claude Code |
| `CLAUDE.md.snippet.md` | Consumer-facing rules (paste into other projects' `CLAUDE.md`) |
| `DISTRIBUTION.md` | Release / hosting mechanics |

## Releasing changes

Releases are label-driven and run in **one phase** — one PR, one merge, one workflow run. The author does **not** run `build.sh` or tag.

1. Branch from `main`, edit `tokens.css` / `components.css`.
2. Add entries under `## [Unreleased]` in `CHANGELOG.md`.
3. Open a PR.
4. Label the PR with **one** of:
   - `release:major` — breaking change (`X+1.0.0`)
   - `release:minor` — new feature, backward-compatible (`X.Y+1.0`)
   - `release:patch` — fix only (`X.Y.Z+1`)
   - `release:skip` — merge with no release cut (still redeploys Pages)
5. Merge.

On the merge push to `main`, `.github/workflows/release.yml` rebuilds `dist/nsa.css`, deploys the full site to GitHub Pages, and — if the label was `release:major|minor|patch` — computes the next semver from the most recent `v*` tag, creates an annotated tag with the `[Unreleased]` block as the tag message, and creates a GitHub Release with `dist/nsa.css` attached. Nothing is committed back to `main`.

See [`DISTRIBUTION.md`](./DISTRIBUTION.md) for the full release flow and emergency-release procedure.
